# Shamir 秘密分享算法 - Go 实现开发文档

## 1. 项目概述

本项目实现了基于伽罗瓦域的 Shamir 秘密分享算法，支持将秘密分割为多个份额，并从阈值数量的份额中恢复原始秘密。

## 2. 核心功能

- **秘密分割**：将单个秘密分割为 `n` 个份额
- **秘密恢复**：从 `t` 个或更多份额中恢复原始秘密
- **安全性**：基于多项式插值的信息论安全
- **同态性**：支持份额的加法同态操作

#### 性能分析

1. **执行时间**：
   - 对于 100 个秘密，执行时间几乎为 0ms
   - 对于 50000 个秘密，执行时间在 148-172ms 之间
   - 随着秘密个数增加，执行时间线性增长

2. **内存使用**：
   - 内存使用与秘密个数和域大小成正比
   - 最大内存使用约为 11MB（1048573 域大小，1000 个秘密）
   - GC 能够有效回收内存，保持内存使用在合理范围内

3. **GC 行为**：
   - 当秘密个数超过 1000 时，GC 周期开始增加
   - 对于 50000 个秘密，GC 周期在 300 左右
   - 大部分情况下 GC 间隔保持在 1 秒左右，没有出现严重卡顿

4. **域大小影响**：
   - 域大小对执行时间影响较小
   - 即使在 1048573（约 1MB）的域大小下，性能仍然保持良好

## 4. 极限测试结论

在不造成 GC "轻微卡顿"（1 秒一次）的情况下：

- **最大域大小**：1048573（约 1MB）
- **最大秘密个数**：50000
- **性能表现**：
  - 50000 个秘密的处理时间约为 150ms
  - 内存使用峰值约为 11MB
  - GC 周期在 300 左右，间隔保持在 1 秒左右

## 5. 代码优化建议

1. **内存优化**：
   - 预分配切片容量，减少内存分配次数
   - 考虑使用对象池复用临时对象

2. **算法优化**：
   - 对于大规模秘密处理，考虑并行化计算
   - 优化多项式评估和拉格朗日插值算法

3. **性能监控**：
   - 添加运行时性能监控
   - 实现自适应批处理策略，根据系统负载调整处理批次大小

## 6. 安全注意事项

1. **素数选择**：使用足够大的素数确保安全性
