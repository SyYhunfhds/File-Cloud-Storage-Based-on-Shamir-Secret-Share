## 二、条目列表问题分析

### 2.1 根本原因：前端模型字段与 API 不匹配

**后端 API `ItemInfo` 定义**（[2026-05-30.md:119-126](file:///f:/GolangWorkspace/CryptographyDesign/backend/api/2026-05-30.md#L119-126)）：

```go
type ItemInfo struct {
    Filename   string    `json:"filename"`
    Owner      string    `json:"owner"`
    Uploader   string    `json:"uploader"`
    UploadedAt time.Time `json:"uploaded_at"`
    ChangedAt  time.Time `json:"changed_at"`
    // ← 没有 minimum_privilege 字段
}
```

**前端 `ItemInfo` 模型**（[item.dart:27-45](file:///f:/GolangWorkspace/CryptographyDesign/frontendv2/lib/models/item.dart#L27-45)）：

```dart
class ItemInfo {
  final String filename;
  final String owner;
  final String uploader;
  final int minimumPrivilege;  // ← API中没有这个字段！
  final DateTime uploadedAt;
  final DateTime changedAt;
}
```

**结论**：前端模型多了一个 `minimumPrivilege` 字段，导致 JSON 解析失败。

### 2.2 技术细节

`json_serializable` 生成的代码（[item.g.dart:27](file:///f:/GolangWorkspace/CryptographyDesign/frontendv2/lib/models/item.g.dart#L27)）：

```dart
minimumPrivilege: (json['minimum_privilege'] as num).toInt(),
```

当 API 响应中没有 `minimum_privilege` 字段时：
- `json['minimum_privilege']` 返回 `null`
- `null as num` 抛出 `TypeError`
- 整个 `ItemInfo.fromJson()` 解析失败

### 2.3 修复方案

**方案一：移除前端多余字段**（推荐）

修改 `frontendv2/lib/models/item.dart`，移除 `minimumPrivilege` 字段及相关代码：

```dart
@JsonSerializable()
class ItemInfo {
  final String filename;
  final String owner;
  final String uploader;
  final DateTime uploadedAt;
  final DateTime changedAt;

  ItemInfo({
    required this.filename,
    required this.owner,
    required this.uploader,
    required this.uploadedAt,
    required this.changedAt,
  });

  factory ItemInfo.fromJson(Map<String, dynamic> json) =>
      _$ItemInfoFromJson(json);
}
```

移除 `privilegeText` getter。

**方案二：将字段设为可选**

```dart
@JsonKey(name: 'minimum_privilege')
final int? minimumPrivilege;

String get privilegeText {
  return minimumPrivilege?.toString() ?? '未知';
}
```

## 三、文件提交问题分析

### 3.1 根本原因

`FilePicker` 在 Windows 桌面平台上不会自动加载文件内容到内存，`file.bytes` 始终为 null。

需要通过 `file.path` + Dart `File` 类手动读取文件。

### 3.2 修复方案

修改 [audit_page.dart](file:///f:/GolangWorkspace/CryptographyDesign/frontendv2/lib/pages/audit_page.dart)：

```dart
import 'dart:io';  // 添加导入

Future<void> _handleUpload() async {
  try {
    final result = await FilePicker.platform.pickFiles();

    if (result != null && result.files.isNotEmpty) {
      final file = result.files.single;
      List<int> bytes;

      if (file.bytes != null) {
        bytes = file.bytes!;
      } else if (file.path != null) {
        bytes = await File(file.path!).readAsBytes();
      } else {
        if (mounted) {
          ScaffoldMessenger.of(context).showSnackBar(
            const SnackBar(content: Text('无法获取文件')),
          );
        }
        return;
      }

      setState(() => _isUploading = true);
      final response = await _apiService.submitItem(bytes, file.name);
      setState(() => _isUploading = false);

      if (response.isSuccess && response.data != null) {
        _showRecoveryCodeDialog(response.data!);
        _loadItems();
      } else {
        if (mounted) {
          ScaffoldMessenger.of(context).showSnackBar(
            SnackBar(content: Text(response.message)),
          );
        }
      }
    }
  } catch (e) {
    setState(() => _isUploading = false);
    if (mounted) {
      ScaffoldMessenger.of(context).showSnackBar(
        SnackBar(content: Text('选择文件失败: $e')),
      );
    }
  }
}
```
