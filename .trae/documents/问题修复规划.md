## 二、详细问题分析

### 2.1 Share字段类型错误

**问题描述**：

```
[ERROR:flutter/runtime/dart_vm_initializer.cc(40)] Unhandled Exception: type 'Share' is not a subtype of type 'Map<String, dynamic>'
```

**根本原因**：

在 `share_page.dart` 的 `_loadShares()` 方法中，对已经反序列化为 `Share` 对象的 `response.data` 再次调用 `Share.fromJson(item)` 进行解析，导致类型不匹配错误。

**用户明确要求**：

> Share 字段必须也是 `String` 类型，它不是 `Map<String, dynamic>` 类型，它就是一整个 Base64 编码的 JSON 序列化字节，桌面端只需要把拿到的原始信息再次加密保存起来即可，**禁止解析禁止解析禁止解析**

**修复方案**：

移除 `share_page.dart` 第 41-42 行中多余的 `Share.fromJson()` 调用，直接使用 `response.data` 返回的 `List<Share>` 数据。

**相关文件**：

- `f:\GolangWorkspace\CryptographyDesign\frontendv2\lib\pages\share_page.dart`

### 2.2 登出API路径404错误

**问题描述**：

请求 `POST /protected/auth/logout` 返回 404，实际应为 `POST /v1/protected/auth/logout`。

**根本原因**：

在 `constants.dart` 中定义的 API 路径常量以 `/` 开头，Dio 将其当作绝对路径处理，跳过了 `baseUrl` 中的 `/v1` 前缀，导致实际请求 URL 缺少版本号。

**修复方案**：

修改 `f:\GolangWorkspace\CryptographyDesign\frontendv2\lib\utils\constants.dart` 中所有 API 路径常量，去掉开头的 `/` 字符。

**修改示例**：

```dart
// 修改前
static const String apiLogout = '/protected/auth/logout';

// 修改后
static const String apiLogout = 'protected/auth/logout';
```
