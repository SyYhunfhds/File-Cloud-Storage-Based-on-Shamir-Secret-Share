# Flutter桌面端更新规划

## 1. 变更分析

### 1.1 认证机制变更
| 项目 | 旧方案 | 新方案 |
|------|--------|--------|
| Token获取 | 登录响应通过Set-Cookie自动设置 | 登录响应在data字段返回 |
| Token携带 | 自动携带Cookie | 仍使用Cookie，但需要前端手动保存并设置 |
| Token存储 | SharedPreferences存Cookie | SharedPreferences存JWT（前端自行管理Cookie） |

**重要说明**: 后端仍然是从Cookie中取出Token进行验证的。前端需要:
1. 登录成功后，从响应的`data`字段获取JWT Token
2. 将Token保存到本地
3. 后续请求时，在请求头中设置`Cookie: authorization=token值`

### 1.2 API路径变更
| API | 旧路径 | 新路径 |
|-----|--------|--------|
| 登出 | `/v1/auth/logout` | `/v1/protected/auth/logout` |
| 用户信息 | `/v1/user/me` | `/v1/protected/user/me` |

### 1.3 用户模型变更
**新增字段**:
- `job`: 职位
- `privilege`: 权限等级
- `registeredAt`: 注册时间

### 1.4 新增API
| API | 方法 | 功能 |
|-----|------|------|
| `/v1/protected/item/submit` | POST | 提交审计条目(multipart) |
| `/v1/protected/item/list` | GET | 分页获取条目列表 |
| `/v1/protected/item/update` | POST | 更新条目信息 |

### 1.5 条目上传流程（重要）
后端设计说明:
1. 前端上传**原始未加密的文件**
2. 后端接收文件后，生成随机密钥
3. 后端使用随机密钥加密文件
4. 后端保存加密后的文件
5. 后端将文件的**Device Share**（通过Shamir秘密共享生成的份额）发给前端

前端职责:
- 用户选择文件后，传递原始文件数据
- 后端返回`share`(Device Share)和`recovery_code`
- 前端需要安全存储Device Share
- 前端需要提示用户保存RecoveryCode

### 2.3 重构API服务
**文件**: `lib/services/api_service.dart`

**变更**:

#### 2.3.1 修改Cookie处理逻辑
```dart
// 不再使用CookieManager自动管理
// 需要手动在请求前设置Cookie头
_dio.interceptors.add(InterceptorsWrapper(
  onRequest: (options, handler) async {
    final token = _prefs.getString(Constants.storageTokenKey);
    if (token != null) {
      // 手动设置Cookie头
      options.headers['Cookie'] = 'authorization=$token';
    }
    print('Request: ${options.method} ${options.path}');
    handler.next(options);
  },
));
```

#### 2.3.2 更新登录方法
```dart
Future<ApiResponse<String>> login({
  required String username,
  required String password,
}) async {
  try {
    final response = await _dio.post(
      Constants.apiLogin,
      data: {
        'username': username,
        'password': password,
      },
    );

    // 登录成功后，保存Token
    if (response.data['code'] == 0) {
      final token = response.data['data'] as String;
      await _prefs.setString(Constants.storageTokenKey, token);
    }

    return ApiResponse<String>(
      code: response.data['code'] ?? 0,
      message: response.data['message'] ?? '',
      data: response.data['code'] == 0 ? response.data['data'] : null,
    );
  } catch (e) {
    return ApiResponse<String>(
      code: ErrorCode.internalError,
      message: '登录失败: ${e.toString()}',
    );
  }
}
```

## 5. 注意事项

### 5.1 Cookie设置方式
```dart
// 在请求拦截器中手动设置Cookie
options.headers['Cookie'] = 'authorization=$token';
```

### 5.2 Token过期处理
- 响应code为66时，表示Token过期或未登录
- 需要引导用户重新登录

### 5.3 文件上传要点
- `POST /v1/protected/item/submit`使用multipart/form-data
- 文件名固定为`item`
- 传递**原始未加密的文件**，后端负责加密

### 5.4 RecoveryCode安全
- 用户提交成功后返回`recovery_code`
- 需要提示用户自行保存
- 退出页面后自动清除内存中的数据
