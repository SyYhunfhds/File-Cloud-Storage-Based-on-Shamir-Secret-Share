# 份额管理页面重构规划

## 一、核心问题确认

根据用户反馈：

1. **份额不是JSON对象**：后端下发的份额是一个**已Base64编码的字符串**，不是多维JSON列表
2. **禁止JSON解析**：桌面端绝对不能尝试任何JSON解析，只需要保存字符串
3. **需要加密存储**：用对称加密密钥保存份额，现在可以硬编码密钥（打TODO）

## 二、修改方案

### 2.1 简化份额存储结构

创建简单的数据类，只存储必要信息：

```dart
// 简单的份额存储结构
class LocalShare {
  final String filename;
  final String shareValue;  // Base64编码的份额字符串
  final DateTime createdAt;

  LocalShare({
    required this.filename,
    required this.shareValue,
    required this.createdAt,
  });

  // 简单的序列化/反序列化
  Map<String, dynamic> toJson() => {
    'filename': filename,
    'shareValue': shareValue,
    'createdAt': createdAt.toIso8601String(),
  };

  factory LocalShare.fromJson(Map<String, dynamic> json) => LocalShare(
    filename: json['filename'],
    shareValue: json['shareValue'],
    createdAt: DateTime.parse(json['createdAt']),
  );
}
```

### 2.2 加密存储服务

使用对称加密保存份额：

```dart
// TODO: [安全改进] 请使用更安全的密钥存储方式（如设备密钥链）
const String _encryptionKey = 'temporary-encryption-key-32-bytes!';

// 加密保存份额列表
Future<void> saveShares(List<LocalShare> shares) async {
  final jsonString = jsonEncode(shares.map((s) => s.toJson()).toList());
  final encrypted = _encrypt(jsonString);
  await _prefs.setString(Constants.storageSharesKey, encrypted);
}

// 解密获取份额列表
Future<List<LocalShare>> getShares() async {
  final encrypted = _prefs.getString(Constants.storageSharesKey);
  if (encrypted == null) return [];
  final jsonString = _decrypt(encrypted);
  final List<dynamic> list = jsonDecode(jsonString);
  return list.map((item) => LocalShare.fromJson(item)).toList();
}

// 简单的XOR加密（示例）
String _encrypt(String plaintext) {
  return plaintext.split('').map((char) {
    int charCode = char.codeUnitAt(0);
    int keyCode = _encryptionKey.charCodeAt(charCode % _encryptionKey.length);
    return String.fromCharCode(charCode ^ keyCode);
  }).join();
}

String _decrypt(String ciphertext) {
  return _encrypt(ciphertext); // XOR加密可逆
}
```
