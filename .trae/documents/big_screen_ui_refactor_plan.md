# 大屏骨架UI重构计划

## 需求分析
用户要求重做大屏UI，采用"一个大屏骨架 + N个模态框"的设计模式，避免路由跳转问题：

1. 主页面作为大屏骨架，默认显示空状态（未登录）
2. 顶部工具栏右侧放置头像按钮
3. 点击头像弹出下拉菜单（含登录按钮）
4. 点击登录按钮弹出登录模态框

## 技术方案
采用简单的状态管理方式，通过 `StatefulWidget` 管理模态框的显示/隐藏状态。

## 文件结构
```
lib/
├── main.dart              # 入口文件
├── pages/
│   └── main_page.dart     # 大屏骨架主页面
├── widgets/
│   ├── login_modal.dart   # 登录模态框
│   └── user_menu.dart     # 用户菜单下拉组件
├── providers/
│   └── auth_provider.dart # 认证状态管理（保留）
├── services/
│   ├── api_service.dart   # API服务
│   └── storage_service.dart # 存储服务
└── models/
    ├── auth.dart          # 认证相关模型
    └── user.dart          # 用户模型
```

## 实现步骤

### 步骤 1: 创建主页面 (main_page.dart)
- 创建大屏骨架布局
- 顶部工具栏（左侧Logo、右侧头像按钮）
- 主内容区域（显示空状态提示）
- 底部状态栏

### 步骤 2: 创建用户菜单组件 (user_menu.dart)
- 下拉菜单样式
- 登录按钮
- 未登录状态显示

### 步骤 3: 创建登录模态框 (login_modal.dart)
- 模态框样式
- 用户名/邮箱切换
- 表单验证
- 登录按钮和加载状态

### 步骤 4: 更新主入口 (main.dart)
- 移除路由配置
- 直接使用主页面作为根组件

### 步骤 5: 更新认证逻辑
- 在登录模态框中调用API
- 登录成功后更新状态并关闭模态框
- 更新用户菜单显示状态

## 状态管理方案
使用 `StatefulWidget` 的 `setState` 管理：
- `showUserMenu`: 控制用户菜单显示
- `showLoginModal`: 控制登录模态框显示
- `user`: 当前用户信息

## 风险处理
- API调用失败时显示错误提示
- 网络请求时显示加载状态
- 模态框点击外部可关闭

## 测试验证
- 点击头像显示/隐藏菜单
- 点击登录按钮弹出模态框
- 登录成功后更新用户状态
- 未登录时显示空状态
